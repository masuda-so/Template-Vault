# Meta

[日本語版はこちら](README.ja.md)

Store notes about how this vault works, how you want to maintain it, and which conventions you want to keep stable over time.

## What This Folder Does

Use this folder for vault-level operating notes such as:

- naming conventions
- folder rules
- review checklists
- plugin decisions
- migration notes
- changelog-style notes about structure updates

Meta notes are about the system, not the subject matter inside the system.

## Why This Folder Is Useful

- Keeps organizational decisions out of project and reference notes.
- Makes it easier to evolve the vault without losing the reasoning behind changes.
- Gives collaborators a single place to understand workflow conventions.

## How To Get Started

Create notes here when you make a structural decision that you want your future self to remember.

## Usage Example

Example note:

```md
# Folder conventions

## Projects
- Active work only
- One note per project unless the project needs its own folder elsewhere

## References
- Evergreen notes and source summaries

## Review cadence
- Weekly review every Friday
- Monthly review on the last day of the month

> [!note] Governance
> Update this policy whenever folder semantics change, and cross-link to [[README#Usage Examples]].

## Checklist
- [ ] Validate naming rules against current folders
- [ ] Update onboarding links in [[Sandbox/README]]
```
