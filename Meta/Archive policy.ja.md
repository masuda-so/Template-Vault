# アーカイブ方針

Vault 内でノートをアーカイブするタイミングと方法を定義します。

## 基本方針

この Vault では **タグのみ** によるアーカイブ方式を採用します。

- ノートのフロントマターに `#archived` タグを追加する。
- ノートは現在のフォルダにそのまま置く。
- アーカイブ済みノートの量が問題になるまで、専用の `Archives/` フォルダへは移動しない。

これによりウィキリンクが維持され、リンク切れを防ぎ、フォルダ構造の変更も不要になります。

> [!note] この方針の再検討について
> アーカイブ済みノートが検索結果や Bases ビューを汚染し始めた場合は、ファイルを移動するよりも、関連する `.base` ファイルで `status != "archived"` フィルタを使って非表示にすることを検討してください。

## 対象範囲

アーカイブ方針はすべてのノートタイプに適用されます。

| フォルダ | アーカイブする条件 |
| --- | --- |
| `Projects/` | プロジェクトが完了または放棄された |
| `Ideas/` | アイデアが破棄された、またはプロジェクト・リファレンスに昇格した |
| `References/` | ノートが陳腐化し、参照価値がなくなった |
| `Clippings/` | ソースが不要になり、処理もされていない |

`Daily/`・`Weekly/`・`Monthly/` のノートは歴史的記録であるため、アーカイブ不要です。そのまま永続的に保持します。

## アーカイブのトリガー

以下のいずれかに該当する場合にアーカイブします。

- **完了**: プロジェクトまたはアイデアが意図した成果に達した。
- **放棄**: プロジェクトまたはアイデアを意識的に中断し、再開しない。
- **不活動**: 6 ヶ月以上レビューも更新もされておらず、現在の関連性もない。
- **上書き**: 新しいノートが同じトピックをカバーしており、古いノートに独自の価値がない。

## アーカイブ手順

1. ノートを開く。
2. フロントマターの `status` を `archived` に変更する。
3. `tags` リストに `#archived` を追加する。
4. 必要に応じて `archived` 日付フィールドを追加する。

例:

```yaml
---
type: project
status: archived
archived: 2026-04-01
tags:
  - project
  - archived
---
```

## メンテナとコントリビューション

アーカイブ戦略が変更された場合（専用 `Archives/` フォルダの導入や不活動しきい値の調整など）は、このノートを更新してください。
