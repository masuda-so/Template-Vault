# レビュー周期

Vault の日次・週次・月次レビューで何を確認するかのチェックリストです。

## 日次レビュー

毎日の開始時または終了時に実施します。

- [ ] `Daily/` に今日の日次ノートを開く（なければ新規作成）。
- [ ] 今日フォーカスする 1〜3 件を書き出す。
- [ ] `Ideas/` をスキャンして、前日にメモしたが文脈が不足しているものに補足する。
- [ ] 昨日の日次ノートの未完タスクを確認し、持ち越すものを今日のノートに移す。
- [ ] 記憶が薄れる前に、重要な出来事・決定・観察を記録する。

> [!tip] 短く済ませる
> 日次レビューは 5 分以内が目安です。目的はフォーカスとキャプチャであり、細かいプランニングではありません。

## 週次レビュー

週に 1 回、できれば金曜日など固定した曜日に実施します。

- [ ] `Weekly/` に `YYYY-Www` 形式で週次ノートを新規作成する。
- [ ] `Projects/` のすべての進行中プロジェクトを確認し、ステータスを更新する。
- [ ] `Ideas/` をスキャンして各ノートの扱いを決める：昇格・保留・破棄。
- [ ] プロジェクトに昇格できるアイデアを `Projects/` に移す。
- [ ] 先週の週次ノートを確認して未解決のフォローアップがないか見る。
- [ ] アーカイブすべきプロジェクトを特定する（[[Archive policy.ja]] 参照）。
- [ ] 来週の優先事項をまとめる。

> [!tip] 昇格は準備ができてから
> アイデアは少なくとも 2 回の週次レビューを経てからプロジェクトに昇格させましょう。早すぎる昇格は `Projects/` を散らかします。

## 月次レビュー

月に 1 回、できれば月末の最終営業日に実施します。

- [ ] `Monthly/` に `YYYY-MM` 形式で月次ノートを新規作成する。
- [ ] `Projects/` のすべての進行中プロジェクトの進捗と関連性を確認する。
- [ ] 完了または放棄すべきプロジェクトを特定し、アーカイブする。
- [ ] `Projects/` や `Clippings/` から永続的な知識を `References/` に昇格させる。
- [ ] `References/` を確認し、陳腐化してアーカイブすべきノートを特定する。
- [ ] Vault 全体の構造を評価する：現在の利用状況にフォルダ構成が合っているか。
- [ ] [[Folder conventions.ja]] と [[Archive policy.ja]] を確認し、更新が必要なら行う。
- [ ] 来月の最優先事項をまとめる。

> [!note] Vault の健全性チェック
> 実際のワークフローが規約から乖離している場合、月次レビューは [[Folder conventions.ja]] や [[Archive policy.ja]] を更新する適切なタイミングです。
