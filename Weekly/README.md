# Weekly

Use this folder for weekly planning and review notes.

## What This Folder Does

Weekly notes help you connect daily activity with medium-term goals. Typical uses include:

- reviewing open projects
- promoting ideas into action
- planning next actions for the coming week
- recording blockers, wins, and follow-up items

This folder is the operational bridge between [Ideas](../Ideas/README.md), [Projects](../Projects/README.md), and [Monthly](../Monthly/README.md).
It works best when paired with [Daily](../Daily/README.md) for capture and [Monthly](../Monthly/README.md) for higher-level planning.

## Why This Folder Is Useful

- Creates a repeatable rhythm for staying on top of active work.
- Prevents ideas and projects from going stale.
- Gives monthly planning a reliable weekly input.

## How To Get Started

Create one note per week, using a consistent naming scheme such as `2026-W14`.

## Usage Example

Example weekly note:

```md
---
type: weekly
week: 2026-W14
reviewed: 2026-03-31
tags:
  - planning
  - review
---

# Week 14 review

## Active projects
- Refresh vault readmes

## Ideas to promote
- Weekly review prompts

## Blockers
- Need clearer folder conventions

> [!tip] Weekly synthesis
> Promote only high-signal items to [[Projects/README]] and archive the rest.

## Next week
- Finish documentation pass

## Related notes
- [[Daily/README]]
- [[Monthly/README]]
```
