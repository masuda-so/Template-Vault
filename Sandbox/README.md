# Sandbox

Use this folder as the learning and experimentation area of the vault.

## What This Folder Does

Sandbox contains onboarding notes and examples that help new users understand how the vault and Obsidian work:

- guided starting points
- formatting examples
- beginner paths based on prior note-taking experience
- introductory notes about vaults and plugins

Current starting points include [Start here.md](Start%20here.md), [Guides/Get started with Obsidian.md](Guides/Get%20started%20with%20Obsidian.md), and the examples under [Formatting](Formatting).

## Why This Folder Is Useful

- Gives new users a safe place to learn the workflow before changing the rest of the vault.
- Centralizes onboarding material instead of scattering help notes across the repository.
- Makes the template more approachable for people who are new to Obsidian.

## How To Get Started

1. Open [Start here.md](Start%20here.md).
2. Follow the path that matches your experience in [Adventurer](Adventurer).
3. Learn core note syntax in [Formatting/Format your notes.md](Formatting/Format%20your%20notes.md).
4. Continue with [Guides](Guides) when you are ready to create and link your own notes.

If you want to understand the local-first model behind the vault, read [Vault is just a local folder.md](Vault%20is%20just%20a%20local%20folder.md).

## Usage Example

title: Project Alpha
date: 2024-01-15

# Project Alpha

This project aims to [[improve workflow]] using modern techniques.
