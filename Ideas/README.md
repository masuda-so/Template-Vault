# Ideas

Capture early-stage thoughts before they become projects, reference notes, or archived decisions.

## What This Folder Does

Use this folder for notes that are still exploratory:

- rough concepts
- future experiments
- possible writing topics
- process improvements you may want to try later

An idea note should stay lightweight. Once it becomes actionable, move or rewrite it in [Projects](../Projects/README.md). If it becomes durable knowledge, move it to [References](../References/README.md).

## Why This Folder Is Useful

- Separates brainstorming from committed work.
- Makes weekly review easier by keeping unfinished thinking in one place.
- Gives you a low-friction place to capture notes without deciding the final structure first.

## How To Get Started

1. Create one note per idea.
2. Give it a short descriptive title.
3. Add just enough context to revisit it later.
4. Review the folder during your weekly or monthly planning.

Example note:

```md
---
type: idea
status: incubating
created: 2026-03-31
tags:
	- workflow
---

# Weekly review prompts

Test a reusable weekly note template with sections for wins, blockers, and next actions.

## Signals that this should become a project
- I want to implement it this month
- It needs multiple steps
- It affects other notes or folders
```
